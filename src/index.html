    What is a Blockchain?

    If we were to build our voting application on the web, we’d run into a few problems:

    1. The data on the database could be changed it could be counted more than once, or removed entirely.

    2. The source code on the web server could also be changed at any time.

    Instead of having a network, a central server, and a database, the blockchain is a network and a database all in one. 

    A blockchain is a peer-to-peer network of computers, called nodes, that share all the data and the code in the network. So, if you’re a device connected to the blockchain, you are a node in the network, and you talk to all the other computer nodes in the network. You now have a copy of all the data and the code on the blockchain. There are no more central servers. Just a bunch of computers that talk to one another on the same network.

    Student Voting Process

    1) To introduce a method of secure authentication, the voting system will use uniten student  
    record to verify the identity of the student.

    2) Once the student has been verify, an email that has a voting ID (Etherium wallet private key) will be send to the Student.

    3) The Student then Log in to the system by entering the student ID via the metamask extention in google chrome, and proceed to vote the candidate.
     
    4) When a Student has selected a candidate and casts his/hers vote, a function in the smart contract adds one vote to the candidate which was voted for.

    5) All transactions which were received are deployed onto the Ethereium blockchain.
